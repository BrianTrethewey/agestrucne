14 class IndependantProcessGroup( object ):
15  '''
16  Simple convenience wrapper around a set of processes,
17  allowing calls into the instance to add a process,
18  or terminate all processes. This class is motivated
19  by implementing parallell replicate AgeStructureNe-inititated
20  simuPop simulations in a PGGuiSimuPop instance via
21  python's multiprocessing class, which apparently interacts
22  with simuPOP such that the multiprocessing.Pool and
23  multiprocessing.Queue process managers do not create
24  simuPOP objects independant of each other. This
25  necessitated using separately instantiated and started
26  multiprocessing.Process objects. Because the typical
27  case will be to run y replicates using x processes, with
28  y>>x, then we need to manage the set of x processes, replacing
29  those finished with fresh processes.
30 
31 
32  All member processes are assumed to be completely
33  independant of all others, so that any I/O messes
34  caused by calling terminate() will not lock up
35  the parent process
36  '''
