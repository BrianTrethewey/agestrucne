61  Subclass of PGGuiApp builds a gui,

63  objects, then on user command runs a simulation
64  (one or more replicates) based on the parameters
65  supplied by the Input object, and to output files

67 
68  Note that for this class I abandoned my intended original
69  organization of the functionality for all PGGuiApp objects.
70  All subclasses of PGGUIApp objects were to contain an
71  AGPOperation object, in this case aPGOpSimuPop object,
72  which contains the code that actually uses Tiago Antao's
73  code to setup and run a SimuPOP session. However, in
74  implenting replicate runs I encountered errors when
75  trying to target a class method when spawning python
76  multiprocessing.process objects from this object. Further,
77  I found non-independant instantiation of simupop populations
78  even when instantiating new PGOpSimuPop objects constructed
79  in separate python multiprocessing.process instances.
80  Thus, parallel processing of simulation replicates required
81  the use of separate python instances for each replicate.
82  Thus currently (Wed Jul 27 18:35:05 MDT 2016),
83  instead of manipulating its own PGOpSimuPop instances,
84  this class depends on code in the pgutilities.py
85  module, which has a def that makes a system call to
86  instantiate a new python interpreter, and call a script,
87  do_sim_replicate.py, which then calls another def in
88  pgutilities.py, which builds and runs a PGOpSimuPop object.
